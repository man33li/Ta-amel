## Milestone Summary

**Key Decisions:**
- 4 phases for v1.0 (logical grouping: testing → bugs → polish → deploy)
- Testing first (tests catch regressions from subsequent fixes)
- Zustand state testing pattern (direct setState/getState)
- Use startsWith for public path matching in middleware
- Root layout force-dynamic for Supabase client components
- Fixed middleware for Vercel production compatibility

**Issues Resolved:**
- Error boundary was empty stub — fully implemented
- No auth redirect in middleware — added with public path handling
- No optimistic update rollback — implemented with state capture
- Build failures on auth pages — fixed with force-dynamic exports
- Middleware incompatibility with Vercel — fixed for production

**Issues Deferred:**
- TiptapEditor test coverage at 67.74% (jsdom limitations)
- No logout button in Header (user has no UI path to sign out)
- Note page duplicates Supabase calls instead of reusing useNotes

**Technical Debt Incurred:**
- React act() warning in HomePage.test.tsx (cosmetic)
- eslint-disable comment in notes/[id]/page.tsx
- Unused Database type export in types/index.ts
- Possibly unused useTheme/useToggleTheme selectors
