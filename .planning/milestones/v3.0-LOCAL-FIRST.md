# v3.0 — Local-First (no Supabase, no OpenAI, no Vercel)

## Goal

Eliminate every subscription / paid third-party from the runtime. The app runs end-to-end on the user's machine: SQLite on disk, embeddings via a local ONNX model, single-user passphrase auth, no managed hosting.

## Stack swap

| Layer | v2.0 | v3.0 |
|---|---|---|
| Database | Supabase Postgres | SQLite (`better-sqlite3`) — one file at `./data/mindforge.db` (configurable) |
| Vector store | Supabase pgvector | `card_embeddings` BLOB column + JS cosine over a bounded set |
| Embeddings | OpenAI `text-embedding-3-small` | `Xenova/all-MiniLM-L6-v2` via `@xenova/transformers`, 384-dim, ~25MB |
| Memory layer | `mem0ai/oss` | Custom `src/lib/memory/store.ts` — embed → upsert → cosine top-K |
| Auth | Supabase Auth (email + Google OAuth) | Single-user passphrase, bcrypt hash, `iron-session` cookie |
| Hosting | Vercel | `npm run dev` / `npm start` on user's machine; Dockerable |
| Browser → DB | Direct Supabase client | `/api/cards`, `/api/wings`, `/api/rooms` (Node-only routes) |

## Net deps removed

`@supabase/ssr`, `@supabase/supabase-js`, `mem0ai`.

## Net deps added

`better-sqlite3`, `@types/better-sqlite3`, `@xenova/transformers`, `iron-session`, `bcryptjs`.

## What survives unchanged

- The palace UX: `/palace` page, `WingList` / `RoomList` / `CardList` / `PalaceSearch`, `useAppStore` selections.
- `src/lib/memory/merge.ts` — the RRF merge over semantic + keyword + recency lanes is identical.
- `NoteCard`'s opt-in move-to-room control.
- The Tiptap editor.
- The shape of `Card`, `Wing`, `Room`, `SearchHit` types (`user_id` retained as a fixed `'me'` for forward compat).

## Out of scope

- Multi-user / family share: the local-first ethos answer is "run a second instance on a second device". Sync between them is a separate feature.
- Export / import: there's no path to bring v1.0/v2.0 Supabase data in. User picked "fresh start, no migration".
- Encrypted SQLite at rest: passphrase gates the UI but the `.db` file is plaintext on disk. Disk-level encryption (FileVault, BitLocker, LUKS) is out of scope.
- Dockerfile: deferred. `npm start` is enough for v3.0.
