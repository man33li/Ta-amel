# v2.0 Requirements — AI Memory + Spatial UX

## Goal

Move MindForge from a flat list of notes to a navigable memory palace with semantic recall.

## Reference inputs

- **mem0ai/mem0** — TypeScript SDK adopted as a direct dependency (`mem0ai`, with `mem0ai/oss` for self-hosted). Provides the `Memory` class, embeddings via OpenAI, and PGVector against Supabase's built-in extension.
- **MemPalace/mempalace** — Python project; concepts only (wings/rooms/drawers taxonomy, hybrid retrieval, verbatim storage). No code ported.

## Functional requirements

### Spatial taxonomy

- A wing has: name, optional color. A wing belongs to a user.
- A room has: name, optional description, parent wing, owning user.
- A card (existing) gains an optional `room_id` foreign key. Null = unfiled.
- Deleting a wing cascades to its rooms; deleting a room un-files its cards (no cascade delete on cards).

### /palace page

- Three panes: wings (left), rooms in active wing (middle), cards in active room (right).
- Empty states walk the user through create-wing → create-room → place-cards.
- Selections are ephemeral (`activeWingId`, `activeRoomId` in Zustand store, not persisted).

### Memory sync

- After any successful card update, fire `POST /api/memory/sync` with `{ cardId }`.
- Failure must not block the editor or surface to the user.
- `/api/memory/sync` extracts plain text from the Tiptap document, then calls `memory.add` scoped by `userId`, `agentId=wingId`, `runId=roomId`, with `metadata.card_id` for traceback.

### Memory search

- `GET /api/memory/search?q=&wing_id?&room_id?&limit?` runs three lanes in parallel:
  1. Semantic via mem0
  2. Keyword via Supabase ILIKE on title (FTS index already built on title + content text)
  3. Recency: cards updated in the last 14 days
- Results are merged via Reciprocal Rank Fusion (RRF, k=60) and labeled by source.
- The palace search bar debounces input by 300ms and renders source badges.

### Graceful degradation

- If `OPENAI_API_KEY` is missing, the memory adapter is a no-op. Palace search still returns keyword + recency hits.
- If `SUPABASE_DB_URL` cannot be parsed, same fallback.

## Non-functional requirements

- No regressions on existing 114-test baseline. New tests added for: merge logic, `usePalace`, palace components, `NoteCard` move control.
- API routes use `runtime = 'nodejs'` (mem0ai pulls in Node-only deps).
- Coverage stays ≥85%.

## Out of scope

- Knowledge graph with temporal validity windows (mempalace v2.1 candidate).
- LLM-based rerank stage on top of hybrid retrieval.
- Auto-tagging cards into wings/rooms via LLM.
- Mobile-optimized palace view.
- Real-time collaboration on wings/rooms.
