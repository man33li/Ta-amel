<decisions_made>
See `.planning/PROJECT.md` "Key Decisions" table — captures the v3.0/v3.1 rationale.

Newly decided 2026-04-29:
- `globalThis.__mindforge_db_cached` is load-bearing in dev mode (Turbopack instantiates a fresh module per route bundle on first compile). Don't refactor back to a module-local `let cached`.

Newly decided 2026-04-30 (v3.3):
- BM25 over a bounded single-user corpus is computed in-process per query; no inverted index needed at this scale.
- Knowledge-graph relations cascade on card delete (`source_card_id ON DELETE CASCADE`). Deleting a card removes the derived knowledge.
- `card_tags` PK is `(card_id, tag, source)` so an auto-tag and a user-tag with the same name coexist.
- Local LLM is opt-in. All `/api/llm/*` routes return 503 `llm_disabled` when the toggle is off; nothing in the rest of the app calls Ollama.
- MCP server runs as a separate stdio process spawned by the client (Claude Desktop, etc.); it shares the same encrypted DB but does not require Next.js.
</decisions_made>
