<task type="auto">
  <name>Task 1: Configure and deploy to Vercel</name>
  <files>vercel.json</files>
  <action>
Create `vercel.json` with framework preset configuration:

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "framework": "nextjs"
}
```

Then attempt deployment using Vercel CLI:

```bash
npx vercel --yes
```

If Vercel CLI is not authenticated, create a checkpoint for the user to:
1. Run `npx vercel login`
2. Then re-run deployment

If deployment succeeds, capture the preview URL.

If Vercel CLI is not installed, install it: `npm install -g vercel` then retry.

Do NOT set env vars via CLI — the user must set NEXT_PUBLIC_SUPABASE_URL and
NEXT_PUBLIC_SUPABASE_ANON_KEY in the Vercel dashboard manually (they contain secrets).
  </action>
  <verify>Deployment completes and returns a preview URL. Run `curl -s -o /dev/null -w "%{http_code}" {preview-url}` — expect 200 or 307 (redirect to login).</verify>
  <done>Application is deployed to Vercel with a working preview URL.</done>
</task>
