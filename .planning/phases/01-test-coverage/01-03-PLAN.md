  <action>
Create integration tests for TiptapEditor:

**Important:** Tiptap uses `immediatelyRender: false` for SSR, which means the editor loads async. Tests need to handle this.

1. Test rendering:
   - Component renders without crashing
   - Shows loading skeleton before editor loads (optional - may be too fast in tests)

2. Test props:
   - Custom placeholder text is applied
   - Initial content is rendered

3. Test callbacks:
   - onUpdate is called when editor content changes

4. Test toolbar:
   - Toolbar buttons are present (Bold, Italic, H1, H2, etc.)

Pattern:
```typescript
import { describe, it, expect, vi } from 'vitest'
import { render, screen, waitFor } from '@testing-library/react'
import userEvent from '@testing-library/user-event'
import { TiptapEditor } from '@/components/notes/TiptapEditor'

const mockOnUpdate = vi.fn()

it('renders without crashing', async () => {
  render(
    <TiptapEditor 
      content={{}} 
      onUpdate={mockOnUpdate} 
    />
  )
  
  // Wait for editor to initialize (async due to immediatelyRender: false)
  await waitFor(() => {
    expect(screen.getByRole('textbox')).toBeInTheDocument()
  }, { timeout: 2000 })
})

it('renders toolbar buttons', async () => {
  render(
    <TiptapEditor 
      content={{}} 
      onUpdate={mockOnUpdate} 
    />
  )
  
  await waitFor(() => {
    expect(screen.getByTitle('Bold (Ctrl+B)')).toBeInTheDocument()
    expect(screen.getByTitle('Italic (Ctrl+I)')).toBeInTheDocument()
    expect(screen.getByTitle('Heading 1')).toBeInTheDocument()
  }, { timeout: 2000 })
})

it('calls onUpdate when content changes', async () => {
  const user = userEvent.setup()
  render(
    <TiptapEditor 
      content={{}} 
      onUpdate={mockOnUpdate} 
    />
  )
  
  await waitFor(() => {
    expect(screen.getByRole('textbox')).toBeInTheDocument()
  })
  
  const editor = screen.getByRole('textbox')
  await user.click(editor)
  await user.type(editor, 'Hello world')
  
  expect(mockOnUpdate).toHaveBeenCalled()
})
```

**Note:** Tiptap in jsdom may have limitations. If tests are flaky, simplify to:
- Render test (smoke test)
- Toolbar button presence test
- Skip typing tests if Tiptap contenteditable has issues in jsdom
