  <name>Task 1: Add optimistic update rollback to useNotes</name>
  <files>src/lib/hooks/useNotes.ts</files>
  <action>
Refactor all three mutation methods in useNotes to use true optimistic updates with rollback:

**createNote:**
1. Save `prev` state before mutation: `const previousNotes = notes`
2. Create a temporary optimistic note with a temp ID (e.g., `crypto.randomUUID()`)
3. `setNotes(prev => [optimisticNote, ...prev])` — immediately show in UI
4. Make server call
5. On success: replace temp note with server-returned note (swap by temp ID)
6. On error: `setNotes(previousNotes)` — rollback to saved state, `setError(message)`

**updateNote:**
1. Save `prev` state: `const previousNotes = notes`
2. `setNotes(prev => prev.map(n => n.id === id ? { ...n, ...updates } : n))` — optimistic
3. Make server call
4. On success: replace with server data (already done)
5. On error: `setNotes(previousNotes)` — rollback, `setError(message)`

**deleteNote:**
1. Save `prev` state: `const previousNotes = notes`
2. `setNotes(prev => prev.filter(n => n.id !== id))` — optimistic remove
3. Make server call
4. On error: `setNotes(previousNotes)` — rollback (note reappears), `setError(message)`

Important: Use the `notes` snapshot from closure, NOT the `prev` callback arg for rollback, to restore exact prior state. Actually, capture via `const previousNotes = [...notes]` at function start.

Keep the existing function signatures and return types unchanged.
