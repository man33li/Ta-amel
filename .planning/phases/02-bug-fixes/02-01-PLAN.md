<task type="auto">
  <name>Task 1: Fix middleware to redirect unauthenticated users</name>
  <files>src/middleware.ts</files>
  <action>
Update middleware.ts to check authentication status after refreshing the session. If `getUser()` returns no user AND the request path is NOT an auth page (/login, /signup, /auth/callback), redirect to /login.

Specific logic:
1. After `await supabase.auth.getUser()`, capture the user: `const { data: { user } } = await supabase.auth.getUser()`
2. Define public paths: `/login`, `/signup`, `/auth/callback`
3. If no user AND path is not public → `return NextResponse.redirect(new URL('/login', request.url))`
4. If user exists AND path is /login or /signup → `return NextResponse.redirect(new URL('/', request.url))` (redirect logged-in users away from auth pages)
5. Otherwise return the existing response

Do NOT change the matcher config or cookie handling — those are correct.
  </action>
  <verify>
Run existing middleware tests: `npx vitest run src/__tests__/middleware` (if they exist).
Manual verification: The middleware logic should be inspectable — protected paths redirect, auth paths don't.
  </verify>
  <done>Middleware redirects unauthenticated users from protected routes to /login, and redirects authenticated users from /login and /signup to /</done>
</task>
